# 修复 Next.js 404 错误

## 问题
页面出现 404 错误，无法加载静态资源（main-app.js, layout.css 等）

## 解决方法

### 方法一：清理缓存并重启（推荐）

1. **停止当前服务**
   - 如果服务正在运行，按 `Ctrl+C` 停止

2. **清理缓存**
   ```bash
   cd Web/webapp
   rm -rf .next
   rm -rf node_modules/.cache
   ```

3. **重新启动**
   ```bash
   npm run dev
   ```

4. **清除浏览器缓存**
   - 按 `Ctrl+Shift+R` (Windows/Linux) 或 `Cmd+Shift+R` (Mac) 强制刷新

### 方法二：使用脚本快速修复

运行修复脚本：
```bash
cd Web/webapp
./fix-404.sh
```

### 方法三：完全重新安装（如果上面方法不行）

```bash
cd Web/webapp
rm -rf .next
rm -rf node_modules
rm -rf .swc
npm install
npm run dev
```

## 常见原因

1. **开发服务器未启动** - 确保 `npm run dev` 正在运行
2. **缓存损坏** - 清理 `.next` 目录
3. **端口冲突** - 确保 3000 端口没有被占用
4. **构建问题** - 重新构建项目

## 检查服务是否运行

```bash
# 检查端口 3000 是否被占用
lsof -i :3000

# 如果被占用，可以：
# 1. 停止占用端口的进程
# 2. 或者使用其他端口：npm run dev -- -p 3001
```
