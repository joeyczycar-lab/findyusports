# 在模拟器里看本地更新

## 为什么模拟器里是旧界面？

App 里有一个「加载地址」配置。  
- 不设置时：加载 **线上站** `https://findyusports.com`（你看到的是旧版）  
- 设置成开发地址后：加载 **本机 3000 端口**（才能看到你刚改的新版）

每次执行 **`cap sync`** 或 **`npm run cap:build:android`**，都会根据当前环境重新生成这个配置。  
如果当时没有带「开发地址」，配置会变回线上，模拟器里就又会变成旧版。

---

## 正确步骤（每次都要按顺序做）

### 1. 开本地网站（必须一直开着）

在项目根目录（webapp）下：

```bash
npm run dev
```

看到 `Ready on http://localhost:3000` 后**不要关**这个终端。

---

### 2. 让 App 使用「开发地址」并跑在模拟器

**方式 A（推荐，一条命令搞定）：**

在 **另一个终端** 里：

```bash
cd /Users/Zhuanz/Documents/findyusports/Web/webapp
npm run cap:run:android:dev
```

会先执行「开发用同步」（把加载地址改成 `http://10.0.2.2:3000`），再在模拟器里安装并打开 App。

**方式 B（用 Android Studio 点 Run）：**

1. 先做一次「开发用同步」（否则会又连回线上）：
   ```bash
   cd /Users/Zhuanz/Documents/findyusports/Web/webapp
   npm run cap:sync:dev
   ```
2. 再在 Android Studio 里点绿色 **Run**，选模拟器运行。

---

### 3. 确认是新版

打开 App 后应该是**新首页**：

- 顶栏：**Logo + 圆角搜索「搜索场地」+ 菜单**（没有「+ 添加场地」、没有第二行搜索）
- 分类：**全部 | 篮球 | 足球**
- **精选场地** 横向滑动
- **APP 专属** 三张卡片
- **发现更多** 深色块
- 底部：**首页 / 探索 / 添加 / 我的**

若还是旧版（大图「不负每一片热爱」、顶栏有「+ 添加场地」），请检查：

1. **`npm run dev` 是否一直在跑**（关掉就会连不上，可能看到空白或错误）
2. 是否用了 **`cap:run:android:dev`** 或先执行了 **`cap:sync:dev`** 再 Run，**不要**只跑 `cap sync` 或 `cap:build:android` 就去看模拟器

---

### 4. 以后要打正式包（连回线上）

打正式包或想用线上站时，再执行一次普通同步即可：

```bash
npm run cap:sync
```

之后在模拟器里跑的就会重新连到 `https://findyusports.com`。
