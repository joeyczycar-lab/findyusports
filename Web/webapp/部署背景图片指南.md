# 部署背景图片到生产环境指南

## 问题
在本地 `http://localhost:3000` 能看到背景图片，但在生产环境 `findyusports.com` 看不到。

## 原因
图片文件可能没有被提交到 Git，或者没有被 Vercel 部署。

## 解决步骤

### 1. 检查图片文件是否在 Git 中

在项目根目录运行：

```bash
cd F:\Findyu
git status Web/webapp/public/hero-background.jpg
```

如果显示 `Untracked files` 或 `Changes not staged`，说明文件未提交。

### 2. 添加并提交图片文件

```bash
# 添加文件到 Git
git add Web/webapp/public/hero-background.jpg

# 提交更改
git commit -m "添加主页背景图片 hero-background.jpg"

# 推送到 GitHub
git push origin master
```

### 3. 验证文件已提交

在 GitHub 上检查：
1. 访问你的仓库：https://github.com/joeyczycar-lab/findyusports
2. 进入 `Web/webapp/public/` 目录
3. 确认 `hero-background.jpg` 文件存在

### 4. 等待 Vercel 自动部署

- Vercel 会自动检测到 GitHub 的推送
- 自动触发新的部署
- 等待部署完成（通常 1-3 分钟）

### 5. 验证生产环境

部署完成后：
1. 访问：https://findyusports.com/hero-background.jpg
   - 如果能看到图片，说明文件已部署成功
   - 如果 404，说明文件未部署

2. 访问：https://findyusports.com
   - 检查主页背景图片是否显示

## 如果还是看不到

### 检查 1：文件大小
如果图片文件太大（> 1MB），可能需要优化：
- 压缩图片
- 或使用 Git LFS

### 检查 2：Vercel 构建日志
1. 登录 Vercel
2. 查看最新的部署日志
3. 检查是否有错误信息

### 检查 3：浏览器缓存
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 或使用无痕模式访问

### 检查 4：直接访问图片 URL
访问：https://findyusports.com/hero-background.jpg
- 如果能看到图片，说明文件已部署
- 如果看不到，说明文件未部署或路径错误

## 快速检查脚本

运行 `检查并提交图片.bat` 来自动检查文件状态。
